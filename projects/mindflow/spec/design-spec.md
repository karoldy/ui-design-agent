## 1. 设计目标与原则

### 1.1 设计目标

| 目标 | 衡量标准 | 对应产品北极星 |
|------|---------|--------------|
| 新建笔记零摩擦 | 从点击「新建」到完成首字输入 < 2s | DAN 提升 |
| 认知负荷最小化 | 页面可视信息密度 < 60%（留白充分） | 留存率 |
| 知识连接直观化 | 用户首次使用 [[ 链接的体验无需外部教程 | 双向链接使用率 |
| 搜索即调用 | 搜索入口始终可见，结果 < 500ms 呈现 | 搜索使用率 |

### 1.2 设计原则

| 原则 | 说明 | 体现方式 |
|------|------|---------|
| **速度即功能** | 每个交互环节都必须最快路径可达 | 快捷键优先、常驻搜索、全局快速新建 |
| **渐进揭示** | 功能复杂度随用户使用深度逐步暴露 | 新手只看到笔记 + 搜索；标签、图谱、链接在探索中浮现 |
| **内容优先** | 任何 UI 元素不应与笔记内容争抢视觉注意力 | 低饱和背景色、最小化工具栏、内容区域 > 60% 视口宽度 |
| **状态可见** | 系统状态（保存、同步、网络）始终可感知但不打扰 | 非侵入式底部状态栏、自动保存动画 |
| **一致优于新奇** | 遵循平台惯例，非必要不创新交互模式 | 采用标准列表、工具栏、侧边栏布局模式 |

## 2. 视觉方向

### 2.1 色彩体系

基于「干净、专注、低视觉噪声」的定位，采用中性色为主色调 + 低饱和强调色点缀的方案。

#### 主色调

```
--color-bg-primary:    #FAFAFA   (页面背景，极浅灰白)
--color-bg-secondary:  #F5F5F5   (侧边栏、卡片、面板背景)
--color-bg-tertiary:   #EEEEEE   (悬停态、列表项选中背景)
--color-bg-inverse:    #1E1E1E   (暗色模式主背景，备用)
```

> 理由：纯白色 #FFFFFF 与深色内容对比度过高，长期阅读易疲劳；极浅灰 #FAFAFA 降低眩光感。侧边栏与主内容区通过轻微色差区分层级。

#### 文字色

```
--color-text-primary:    #1A1A1A   (正文内容)
--color-text-secondary:  #6B6B6B   (辅助信息、时间戳、描述)
--color-text-tertiary:   #9E9E9E   (占位符、禁用态)
--color-text-link:       #3B82F6   ([[ 双向链接锚点)
--color-text-on-accent:  #FFFFFF   (强调色上文字)
```

#### 边框色

```
--color-border: #E8E8E8   (边框、分割线)
```

#### 强调色

```
--color-accent:      #3B82F6   (统一强调色 — 链接、选中态、激活态)
--color-accent-hover:#2563EB   (强调色悬停态)
--color-accent-soft: #E8F0FE   (强调色弱背景 — 标签、选中背景)
```

> 理由：单强调色方案降低视觉复杂度。#3B82F6 为蓝色系，在不同平台上均传达「可交互/已激活」信号，且与 Google/Microsoft 生态的蓝色强调色习惯一致。

#### 语义色

```
--color-success:     #10B981   (同步完成、保存成功)
--color-warning:     #F59E0B   (离线状态、同步冲突)
--color-error:       #EF4444   (同步失败、操作错误)
```

#### 标签色（预设 8 色，用户可自定义）

```
--tag-red:     #FEE2E2
--tag-orange:  #FFEDD5
--tag-yellow:  #FEF3C7
--tag-green:   #D1FAE5
--tag-teal:    #CCFBF1
--tag-blue:    #DBEAFE
--tag-purple:  #EDE9FE
--tag-pink:    #FCE7F3
```

> 理由：标签色使用弱背景（pastel 系），避免在笔记列表和编辑器内与正文产生视觉冲突。用户如需强调可配合标签文字加粗。

#### 标签点缀色（vivid 色，配套标签圆点）

```
--dot-red:    #EF4444
--dot-orange: #F97316
--dot-yellow: #EAB308
--dot-green:  #22C55E
--dot-teal:   #14B8A6
--dot-blue:   #3B82F6
--dot-purple: #8B5CF6
--dot-pink:   #EC4899
```

> `--dot-*` 为标签圆点的 vivid 色，与 `--tag-*` pastel 背景色配套使用，在侧边栏标签树和笔记列表标签圆点中使用。

### 2.2 字体体系

| 用途 | 字体 | 后备 | 理由 |
|------|------|------|------|
| UI 标签 & 正文 | Noto Sans (400/500/600/700) | sans-serif | 全局统一字体，跨语言一致性好，CJK 与 Latin 混排美观 |
| 代码块 / 内联代码 | JetBrains Mono / SF Mono | "Cascadia Code", "Fira Code", monospace | 代码等宽展示，清晰区分 |

**CDN 引入**：
```html
<link href="https://fonts.googleapis.com/css2?family=Noto+Sans:wght@400;500;600;700&display=swap" rel="stylesheet">
```

**CSS 字体栈**：
```css
font-family: 'Noto Sans', sans-serif;
```

> 理由：Noto Sans 作为 Google 开源字体，覆盖广泛的 Unicode 字符集，中英文混排效果优秀，视觉风格干净中性，符合「快速、干净、专注」的产品定位。

#### 字号系统

```
--font-size-xs:   11px   (元数据、时间戳)
--font-size-sm:   13px   (辅助文字、侧边栏项)
--font-size-base: 15px   (正文、笔记列表标题)
--font-size-lg:   18px   (h4、小节标题)
--font-size-xl:   22px   (h3)
--font-size-2xl:  28px   (h2)
--font-size-3xl:  36px   (h1)
```

> 理由：正文使用 15px 而不是常见的 14px/16px：14px 在阅读长文本时偏小，16px 在笔记列表中信息密度降低；15px 是折中值，兼顾阅读舒适度和信息密度。

### 2.3 间距系统

采用 4px 为基础单位的 4n 间距体系，符合 Web 和 iOS 主流设计系统惯例。

```
--space-1:  4px
--space-2:  8px
--space-3:  12px
--space-4:  16px
--space-5:  20px
--space-6:  24px
--space-8:  32px
--space-10: 40px
--space-12: 48px
--space-16: 64px
```

#### 关键间距约定

| 场景 | 间距值 | 说明 |
|------|--------|------|
| 侧边栏内边距 | 12px (space-3) | 紧凑但不拥挤 |
| 笔记列表项 padding | 16px 20px (space-4 space-5) | 垂直 16px，水平 20px |
| 编辑器内容区 | 40px 左右 (space-10) | 留出呼吸空间 |
| 段落间距 (Markdown) | 16px (space-4) | 阅读体验舒适 |
| 组件间距（相邻） | 8px (space-2) | 标准组件间距 |
| 模块间距（分区） | 24px (space-6) | 明确区块边界 |

### 2.4 圆角与阴影

#### 圆角

```
--radius-sm:   4px    (按钮、标签、搜索输入框)
--radius-md:   8px    (卡片、模态框、笔记列表项)
--radius-lg:   12px   (大卡片、面板)
--radius-xl:   16px   (弹出菜单、用户头像)
--radius-full: 9999px (圆形元素、徽章)
```

#### 阴影

```
--shadow-sm:   0 1px 2px rgba(0,0,0,0.05)      (轻浮层 — 悬停)
--shadow-md:   0 2px 8px rgba(0,0,0,0.08)      (浮层 — 下拉菜单)
--shadow-lg:   0 8px 24px rgba(0,0,0,0.12)     (模态框、抽屉)
--shadow-xl:   0 16px 48px rgba(0,0,0,0.16)    (全局通知、悬浮面板)
```

> 理由：阴影整体偏浅（左上右下透光隐喻），符合「干净、轻盈」的产品气质。

### 2.5 动效

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| 页面切换 | 淡入淡出 + 轻微上移(8px) | 200ms | ease-out |
| 侧边栏展开/收起 | 滑动 240px | 200ms | ease-in-out |
| 笔记列表新建项出现 | 淡入 + 上移(12px) | 250ms | ease-out |
| 自动保存完成 | 绿点闪现 + 渐变消失 | 600ms | ease-in-out |
| 搜索下拉面板 | 淡入 + 下移 | 150ms | ease-out |
| [[ 链接弹出菜单 | 淡入 + 缩放(0.95→1) | 120ms | ease-out |

> 原则：所有动效时长 < 300ms（搜索面板除外），不做无意义装饰动效。

## 3. 布局结构

### 3.1 全局布局 (Web 桌面端)

```
┌─────────────────────────────────────────────┐
│  [Logo]  [ 全局搜索 ]  [新建] [设置] [头像] │  ← 顶部导航栏 (48px)
├────────┬────────────────────────────────────┤
│        │                                    │
│ 笔记   │                                    │
│ 本树   │       内容主区域                    │
│        │   （笔记列表 / 编辑器 / 图谱）      │
│ 标签   │                                    │
│ 层级   │                                    │
│        │                                    │
│ 回收站 │                                    │
│        │                                    │
├────────┤                                    │
│ 空间   │                                    │
│ 使用量 │                                    │
├────────┴────────────────────────────────────┤
│            状态栏 (同步状态 / 网络)           │  ← 底部状态栏 (28px)
└─────────────────────────────────────────────┘
 |-- 240px --|         |-- 剩余 --|
```

| 区域 | 宽度 | 说明 |
|------|------|------|
| 侧边栏 | 240px (可收起) | 笔记本树 + 标签层级的统一导航 |
| 主内容 | 剩余宽度 (min 640px) | 根据当前路由展示笔记列表 / 编辑器 / 图谱 |
| 顶部导航 | 全宽, 48px | 搜索入口 + 快速操作 + 用户设置 |
| 底部状态栏 | 全宽, 28px | 极简状态指示 |

> 布局决策理由：非对称侧边栏布局是多数笔记产品（Notion、Obsidian、Bear）的选择，用户无需学习。240px 宽度兼顾目录树可见性和内容区空间。

### 3.2 路由结构

```
/                    → 笔记列表（最近修改）
/notes/:id           → 笔记编辑器
/notes/:id/history   → 版本历史（P1）
/search?q=xxx        → 搜索详情页
/graph               → 知识图谱（P1）
/settings            → 设置页
/settings/account    → 账户管理
/settings/sync       → 同步设置
/settings/import     → 数据导入导出
```

### 3.3 布局状态矩阵

| 状态 | 侧边栏 | 搜索栏 | 笔记列表 | 编辑器 | 底部状态栏 | 提示/引导 |
|------|--------|--------|---------|--------|----------|----------|
| 空状态 (无笔记) | 可见 | 可见（置灰） | 空状态引导插画 | 隐藏 | 可见 | 居中引导卡片：「新建第一篇笔记」|
| 列表浏览 | 可见 | 可见 | 显示笔记列表 | 隐藏 (或预览) | 可见 | — |
| 编辑中 | 可收起 | 可见 | 可折叠 | 全屏编辑 | 可见 | — |
| 搜索中 | 可见 | 聚焦 + 展开 | 搜索结果替代列表 | 隐藏 | 可见 | — |
| 离线状态 | 可见 | 可见 | 列表正常 (本地) | 正常编辑 | 黄色「离线」提示 | — |
| 移动端 (窄屏) | 覆盖式抽屉 | 可见（可折叠） | 全宽列表 | 全宽编辑 | 简化 | — |

## 4. 核心页面清单

### 4.1 笔记列表页

**入口**：`/`（默认首页）
**目标**：用户快速找到最近编辑的笔记，或新建笔记。

#### 设计要点
1. **列表排序**：默认按最近修改时间降序，提供下拉筛选（最近修改 / 创建时间 / 标题 A-Z）。
2. **列表项设计**：每项显示 标题 + 前 120 字纯文本摘要 + 标签+ 时间戳（相对时间「3分钟前」）。
   - 标题与摘要之间留 4px 间距
   - 标签以彩色圆点 + 文字形式显示，最多显示 3 个，超出 +N
   - 右侧浮动显示编辑时间或 Pin 图标
3. **快速操作**：鼠标悬停列表项时出现「更多」按钮（⋯），展开菜单包含：重命名、移动、删除、复制链接。
4. **分组头**：按日期分组（今天、昨天、7天内、更早），分组标题使用 secondary 色文字 + 14px。
5. **无限滚动**：列表向下滚动时自动加载更多，避免分页对浏览体验的打断。
6. **拖拽排序**：笔记本内支持拖拽调整列表顺序（具体看 P0/P1 决策）。
7. **批量操作**：支持多选（Cmd+Click 或长按 Shift），批量移动/删除/加标签。
8. **草稿恢复**：退出编辑时若有未保存内容，列表中该笔记项显示「未保存」橙色标记。

#### 空状态（无笔记时）
- 居中显示引导插画（极简线条风格）
- 主标题：「欢迎来到 MindFlow」
- 副标题：「点击下方按钮开始你的第一篇笔记"
- CTA：「新建笔记」（large button）
- 底部可跳过：「导入数据」（P1 功能，但入口可以存在）

### 4.2 Markdown 编辑器

**入口**：点击笔记列表项或全局新建按钮
**目标**：提供快速、专注的写作体验，且不打断思维流。

#### 设计要点
1. **工具栏**：顶部浮动工具栏（当编辑器处于焦点时显示），包含：
   - 格式化：H1-H3 | Bold | Italic | Strikethrough | Code | Blockquote | Divider
   - 插入：Checklist | Bullet List | Ordered List | Image | Link
   - 操作：Focus Mode（隐藏工具栏）、Preview Toggle、More (⋮)
   - 工具栏在不活跃 2 秒后自动淡出（可配置）
2. **编辑器主体**：
   - 全宽 Markdown 编辑区，左侧保留 40px 间距，右侧保留 40px 间距
   - 实时渲染预览（Live Preview）：输入 `# ` 即时渲染为标题样式，输入 `**text**` 即时加粗
   - 光标处显示原始 Markdown 符号，离焦后隐藏
   - 支持块级操作：回车分割块，块左侧显示「⋮⋮」拖拽手柄（悬停时出现）用于拖动排序块
3. **[[ 双向链接**：
   - 输入 `[[` 触发联想搜索下拉菜单
   - 搜索范围：当前空间内所有笔记标题
   - 搜索结果分两组：精确匹配 / 模糊匹配
   - 选中后渲染为蓝色链接样式 `--color-text-link`
   - 已链接笔记在列表中显示「N 个反向链接」指标
4. **# 标签联想**：
   - 输入 `#` 触发标签联想
   - 显示标签层级树：`#设计/UX/交互` 会按 `/` 展开层级
   - 选中后标签渲染为彩色标签样式
5. **自动保存指示**：
   - 底部状态栏显示保存状态：保存中(旋转动画) → 已保存(绿色圆点)
   - 自动保存触发条件：输入停止 2 秒后
6. **Focus Mode**：
   - 点击工具栏 Focus Mode 按钮，隐藏全部 UI（侧边栏、顶部导航、工具栏），仅剩编辑器
   - 按下 Esc 退出
   - 进入时产生平滑动画过渡

#### 编辑器空状态

- 标题行占位符：「无标题笔记」
- 正文占位符：「开始写作… 输入 [[ 链接笔记，输入 # 添加标签」
- 注意：占位符不显示时不影响输入光标位置

### 4.3 侧边栏导航

**目标**：提供笔记本树 + 标签层级两个维度的内容组织导航。

#### 设计要点
1. **分区结构**：
   - **搜索**（顶部常驻，见 4.4 全局搜索）
   - **快捷入口**：所有笔记 / 每日笔记 / 回收站
   - **笔记本**：树状结构，支持折叠/展开
     - 每个笔记本前有文件夹图标 (mdi-folder-outline / mdi-folder-open-outline)
     - 笔记本名后显示笔记数量
     - 拖拽排序支持
   - **标签**：按层级树展示，支持折叠/展开
     - 标签多级缩进 16px
     - 每个标签前有彩色圆点或标签图标
     - 显示该标签下笔记数量
   - **空间使用量**：底部进度条 + 文本
2. **交互**：
   - 笔记本/标签名称上右键打开上下文菜单（重命名、删除、新建子项）
   - 点击笔记本进入该笔记本的笔记列表
   - 点击标签过滤出该标签下的所有笔记
   - 支持多选标签的组合筛选（Cmd+Click 多选）
3. **收起/展开**：侧边栏宽度 240px，支持通过左侧边缘拖拽调整宽度（min 180px, max 360px），或点击顶部 hamburger 图标收起

#### 空状态（无笔记本/标签）
- 显示引导文字：「创建笔记本开始组织你的笔记」
- 底部显示「+ 新建笔记本」按钮

### 4.4 全局搜索

**目标**：始终可访问的搜索入口，< 500ms 返回本地搜索结果。

#### 设计要点
1. **入口**：顶部导航栏中央常驻搜索条（宽度 360px~480px 响应式）
   - 键盘快捷键 `Cmd+K` (Mac) / `Ctrl+K` (Win/Linux) 聚焦搜索
   - 未聚焦时显示占位符：「搜索笔记… (⌘K)」
2. **搜索下拉面板**（聚焦后展开）：
   - 面板以覆盖层形式弹出，不改变页面布局
   - 宽度随搜索条宽度（360px~480px），最大高度 480px（溢出滚动）
   - 结果按类型分组（顶部显示分组标题）
   - **分组**：精确匹配标题 > 模糊匹配标题 > 全文匹配
   - 无结果时显示：「未找到相关笔记」+ 建议：「尝试不同的关键词或检查拼写」
3. **搜索结果项**：
   - 与笔记列表项设计一致（标题 + 摘要 + 标签）
   - 匹配文字高亮（黄色背景 `#FEF3C7`）
   - 键盘上下箭头导航，Enter 打开选中的笔记
4. **高频动作**：搜索条右侧显示「新建包含此关键词的笔记」按钮（+ 图标）
5. **退出**：点击覆盖层外部、按 `Esc`、或搜索结果打开后自动关闭

#### 搜索空状态
- 面板保持打开，显示当前已输入的搜索词
- 无结果时显示空状态插画（极简）和引导文字

### 4.5 知识图谱页 (P1, W14)

**入口**：侧边栏导航底部「知识图谱」入口
**目标**：可视化展示笔记之间的链接关系，辅助知识探索。

#### 设计要点
1. **力导向图**：
   - 节点表示笔记，边表示 [[ 双向链接关系
   - 节点大小反映链接数量（被引用越多的笔记节点越大）
   - 边带有方向箭头（从 A→B 指向 B→A）
   - 默认视图显示当前空间全部笔记的网络图
2. **交互**：
   - 拖拽空白区域平移画布
   - 滚轮缩放
   - 点击节点展示笔记标题悬浮卡，再次点击跳转到笔记编辑页
   - 双击节点聚焦该节点及一跳连接的所有节点
   - 节点拖拽可手动调整布局位置（P1 功能）
3. **控制面板**：
   - 左上角缩放控件（+ / - / 适应屏幕）
   - 右上角图例（节点含义、边含义）
   - 底部搜索框：高亮匹配的节点
4. **空状态**（无任何链接的笔记）：
   - 显示星形分布图（每个笔记是一个独立节点，无连线）
   - 引导文字：「创建 [[ 双向链接来建立笔记之间的联系」
   - 示例：展示两个链接笔记的预览图
5. **性能**：
   - WebGL 渲染（d3-force 或 @neo4j/react-visualization）
   - 节点数 > 500 时自动启用虚拟化（仅渲染视口内的节点）

### 4.6 用户认证

**目标**：安全的身份验证流程，支持 OAuth 和邮箱注册。

#### 设计要点
1. **登录页**：
   - 居中对齐卡片式布局
   - 品牌 logo + tagline
   - 登录方式：Google OAuth / GitHub OAuth / 邮箱 + 密码
   - OAuth 按钮使用对应品牌色（Google 红、GitHub 黑）
   - 底部「注册」链接
2. **注册页**：
   - 与登录页视觉一致
   - 邮箱注册字段：邮箱 / 密码 / 确认密码
   - OAuth 注册一键完成
3. **密码重置**：邮箱验证码流程
4. **生物识别**（移动端 P1）：锁屏后进入 App 时指纹/FaceID 验证

### 4.7 设置页

**目标**：集中的账户与偏好管理。

#### 设计要点
1. **布局**：左侧导航列表 + 右侧内容区的双栏布局（类似系统设置）
   - 左侧 200px 宽
2. **设置项**：
   - **账户**：头像、昵称、邮箱、更改密码、注销
   - **同步**：同步状态显示、手动同步按钮、最近同步时间，账户存储使用量
   - **编辑器**：默认字号、字体选择、行间距、Focus Mode 快捷键、自动保存间隔
   - **外观**：主题（亮色/暗色/跟随系统）、侧边栏默认展开/收起
   - **数据**：导出（Markdown / JSON）、导入、清空所有数据
   - **关于**：版本号、许可证、隐私政策、反馈
3. **交互**：表单修改后自动保存（或底部「保存」按钮）

### 4.8 空状态与引导

**目标**：帮助新用户度过「空白焦虑」，降低首次使用门槛。

#### 空状态场景清单

| 场景 | 位置 | 引导内容 |
|------|------|---------|
| 新用户，0 笔记 | 笔记列表页 | 插画 +「欢迎来到 MindFlow」+ 新建按钮 |
| 笔记本内无笔记 | 笔记列表页（笔记本筛选后） | 「该笔记本还没有笔记」+ 新建按钮 |
| 标签下无笔记 | 笔记列表页（标签筛选后） | 「该标签还没有笔记」|

### 5.3 组件设计原则

1. **组件规格**：每个组件输出设计稿时需覆盖 Default / Hover / Active / Focus / Disabled / Error 状态
2. **组件尺寸一致性**：所有可交互组件最小触控区域 44x44px（WCAG 2.1 要求）
3. **Z-index 层级**：Dropdown(100) → Header(200) → Modal/Overlay(300) → Toast(400) → Tooltip(500)
4. **原子化设计**：复合组件优先使用基础组件组合，不产生冗余样式

## 6. 交互模式

### 6.1 导航模式

| 行为 | 交互 | 反馈 |
|------|------|------|
| 切换笔记 | 点击列表项 | 列表项背景变化 → 编辑器加载 → 平滑过渡 |
| 笔记本切换 | 侧边栏点击笔记本 | 笔记列表更新，Breadcrumb 更新 |
| 标签筛选 | 侧边栏点击标签 | 笔记列表过滤，标签高亮，Breadcrumb 更新 |

| 侧边栏收起 | 点击 hamburger | 侧边栏 200ms 滑出，主内容区平铺扩展 |
| 返回笔记列表 | 浏览器返回 (←) | 保持编辑器滚动位置（如已打开） |

### 6.2 编辑模式

| 行为 | 交互 | 反馈 |
|------|------|------|
| 新建笔记 | Cmd+N / 点击 + 按钮 | 列表顶部新建项（占位标题「无标题笔记」）→ 自动聚焦标题输入 |
| 自动保存 | 输入停止 2s 后 | 底部状态栏「保存中」→「已保存」动画 |
| 手动保存 | Cmd+S | 立即保存，状态栏反馈 |
| 退出编辑 | 点击列表其他项 / 新标签页 | 自动保存后切换 |
| 未保存退出 | 关闭 tab / 离开页面 | 浏览器原生「未保存」提示 |
| [[ 触发 | 输入 `[[` | 下拉菜单弹出（120ms）|
| # 触发 | 输入 `#` | 下拉菜单弹出，展示标签树 |
| Focus Mode | 点击 Focus Toggle | 侧边栏 + 导航 + 工具栏全部隐藏 |
| 块排序 | 拖拽行首手柄 | 块移动动画，重新编号列表 |
| 图片插入 | 拖拽 / 粘贴 / 工具栏 | 图片上传进度显示 |

### 6.3 搜索模式

| 行为 | 交互 | 反馈 |
|------|------|------|
| 聚焦搜索 | Cmd+K / 点击搜索条 | 搜索条高亮，下拉面板展开 |
| 实时搜索 | 输入字符（200ms debounce） | 结果分组实时更新，匹配文字高亮 |
| 键盘导航 | ↑↓ 键移动选中 | 选中行背景变化，滚动跟随 |
| 确定选择 | Enter / 点击结果 | 面板关闭，跳转到笔记编辑页 |
| 新建并搜索 | 点击「+」按钮 | 新建标题为搜索词的笔记，聚焦编辑器 |
| 关闭搜索 | Esc / 点击外部 | 面板关闭，搜索词清空 |

### 6.4 图谱交互 (P1)

| 行为 | 交互 | 反馈 |
|------|------|------|
| 平移 | 拖拽空白 | 画布跟随，平滑惯性 |
| 缩放 | 滚轮 / 捏合 | 以光标为中心缩放 |
| 节点选中 | 点击节点 | 节点高亮 + 悬浮卡显示标题和标签 |
| 节点聚焦 | 双击节点 | 相机过渡到节点及其一跳邻居 |
| 节点跳转 | 节点悬浮卡上点击「打开」 | 跳转到笔记编辑页 |
| 适应屏幕 | 点击 Adapt 按钮 | 画布缩放使所有节点可见 |
| 搜索高亮 | 图谱搜索输入 | 匹配节点加光晕动效 |

### 6.5 通用交互规则

1. **键盘导航**：所有列表支持 ↑↓ 方向键导航，Enter 选中，Tab 移出
2. **快捷键**：所有快捷键应在 UI 上提示（如搜索框内显示 `⌘K`）
3. **右击支持**：侧边栏、笔记列表支持右键上下文菜单
4. **拖拽支持**：侧边栏笔记本排序、笔记列表项排序（P1）
5. **Undo/Redo**：编辑器内支持 Cmd+Z / Cmd+Shift+Z，撤销 50 步
6. **触觉反馈**（移动端）：关键操作（保存、同步成功）伴随触觉反馈
7. **网络感知**：离线时禁用云端同步相关 UI，仅显示本地操作选项
8. **过渡动效**：所有页面/视图切换使用标准过渡（200ms ease-out）

## 7. 响应式策略

### 7.1 断点定义

| 断点 | 宽度 | 设备 | 布局模式 |
|------|------|------|---------|
| Desktop | >= 1024px | 桌面浏览器 | 三栏（侧边栏 + 列表 + 编辑器），或双栏（侧边栏 + 内容）|
| Tablet | 768px - 1023px | iPad 横屏 / 小屏笔记本 | 侧边栏可折叠，默认展开内容区 |
| Mobile | < 768px | 手机 / iPad 竖屏 | 单栏全宽布局，侧边栏为覆盖式抽屉 |

### 7.2 各断点布局

#### Desktop (>= 1024px)

```
默认视图（笔记列表）：
[侧边栏 240px] [笔记列表 320-480px] [空/预览]
  或
[侧边栏 240px] [内容区]

编辑状态：
[侧边栏 240px (可收起)] [编辑器 剩余]
```

#### Tablet (768px - 1023px)

```
默认视图：
[侧边栏 240px (可折叠)] [内容区]

编辑状态：
[编辑器 全宽] — 侧边栏收起
             — 侧边栏作为覆盖式抽屉 (z-index)
```

#### Mobile (< 768px)

```
默认视图：
[顶部导航 (简化)]  ← 只有 Logo + 搜索 + 新建
[笔记列表 全宽]
[底部导航栏]       ← 首页 / 搜索 / 设置

编辑状态：
[编辑器 全宽]
[顶部 mini 工具栏] ← 格式化 + 返回
[底部]            ← 保存状态 + 键盘工具栏
```

### 7.3 响应式设计要点

| 元素 | Desktop | Tablet | Mobile |
|------|---------|--------|--------|
| 侧边栏 | 常驻 240px | 可收起，覆盖式 | 覆盖式抽屉，全宽 |
| 笔记列表 | 宽度 320~480px | 宽度 50%~70% | 全宽 |
| 编辑器 | 居中 680px 最大宽度 | 全宽（留白 24px） | 全宽（留白 16px）|
| 搜索条 | 中央 360~480px | 中央 240~360px | 可折叠（图标触发）|
| 工具栏 | 浮动完整 | 浮动完整 | 简化（基础格式 + 更多）|
| 底部状态栏 | 全宽 | 全宽 | 隐藏（集成到顶部）|
| 知识图谱 | 完整 | 完整（控件缩小） | 全屏 + 底部控件 |
| 设置页 | 双栏 | 双栏（侧栏固定） | 单栏（导航下拉）|
| 空状态引导 | 居中卡片 | 居中卡片 | 全屏居中 |

### 7.4 移动端特殊行为

- **编辑器**：`[[` 和 `#` 触发时拉起底部面板（而非下拉菜单），提供更大的选择区域
- **返回手势**：从左边缘滑动返回笔记列表（iOS 原生手势 + Android 自定义）
- **搜索**：点击搜索图标全屏展开搜索页，结果列表占满屏幕
- **侧边栏**：从左侧边缘滑动拉出
- **底部导航**：固定底部 4 个 Tab：笔记 / 搜索 / 图谱 / 设置，适配移动端单手操作

## 9. 暗色模式

### 9.1 暗色模式色彩 Token

与第 2.1 节亮色模式色彩体系一一对应，定义完整的暗色模式 CSS 变量：

#### 背景色

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-bg-primary` | `#1A1A1A` | 页面背景 |
| `--color-bg-secondary` | `#242424` | 侧边栏、卡片、面板背景 |
| `--color-bg-tertiary` | `#2E2E2E` | 悬停态、列表项选中背景 |
| `--color-bg-inverse` | `#FAFAFA` | 亮色模式在暗色下的反色（极少使用） |

> 理由：深灰黑系背景降低 OLED 屏幕功耗，同时避免纯黑（#000000）导致的内容区对比度过低。通过三层灰度递进区分层级：页面基底最暗，浮层面板稍亮，悬停态进一步提亮。

#### 文字色

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-text-primary` | `#E8E8E8` | 正文内容 |
| `--color-text-secondary` | `#A0A0A0` | 辅助信息、时间戳、描述 |
| `--color-text-tertiary` | `#6B6B6B` | 占位符、禁用态 |
| `--color-text-link` | `#60A5FA` | [[ 双向链接锚点 |
| `--color-text-on-accent` | `#FFFFFF` | 强调色上文字 |

> 理由：正文色使用 #E8E8E8 而非白色，降低高对比度刺眼感。链接色从亮色的 #3B82F6 调亮至 #60A5FA，在深色背景下保证可读性。

#### 强调色

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-accent` | `#60A5FA` | 统一强调色 — 链接、选中态、激活态 |
| `--color-accent-hover` | `#93C5FD` | 强调色悬停态 |
| `--color-accent-soft` | `#1E3A5F` | 强调色弱背景 — 标签、选中背景 |

> 理由：暗色模式下强调色适当提亮，补偿深色背景上的感知亮度损失。弱背景使用深蓝 #1E3A5F 替代亮色的 #E8F0FE，在暗色模式下保持层级感。

#### 语义色

| Token | 色值 | 用途 |
|-------|------|------|
| `--color-success` | `#34D399` | 同步完成、保存成功 |
| `--color-warning` | `#FBBF24` | 离线状态、同步冲突 |
| `--color-error` | `#F87171` | 同步失败、操作错误 |

> 理由：暗色模式下语义色整体提亮一个层次，确保在深色背景上的辨识度。

#### 标签色（暗色适配）

标签背景色从亮色的 pastel 系改为深色系+亮文字：

| Token | 亮色色值 | 暗色色值 |
|-------|---------|---------|
| `--tag-red` | `#FEE2E2` | `#3B1818` |
| `--tag-orange` | `#FFEDD5` | `#3B2212` |
| `--tag-yellow` | `#FEF3C7` | `#3B3018` |
| `--tag-green` | `#D1FAE5` | `#163B22` |
| `--tag-teal` | `#CCFBF1` | `#143B30` |
| `--tag-blue` | `#DBEAFE` | `#182A3B` |
| `--tag-purple` | `#EDE9FE` | `#22183B` |
| `--tag-pink` | `#FCE7F3` | `#3B1830` |

标签圆点缀点色（`--dot-*`）在暗色模式下保持不变（仍为 vivid 色），由于深色背景下 vivid 色辨识度足够。

> 理由：标签背景在暗色中使用深色而保留色调倾向，保证颜色可区分性同时不喧宾夺主。标签文字在暗色中使用 `--color-text-primary`（#E8E8E8）而非亮色模式的深色文字。

#### 边框色

| Token | 色值 |
|-------|------|
| `--color-border` | `#3A3A3A` |

> 理由：暗色背景上边框需要一定对比度才能可见，#3A3A3A 比亮色模式的 #E8E8E8 暗 4 阶，在暗色背景下视觉重量一致。

#### 阴影（暗色适配）

暗色模式下阴影减弱透明度，保持深度质感但不突兀：

```
--shadow-sm:   0 1px 2px rgba(0,0,0,0.30)
--shadow-md:   0 2px 8px rgba(0,0,0,0.35)
--shadow-lg:   0 8px 24px rgba(0,0,0,0.40)
--shadow-xl:   0 16px 48px rgba(0,0,0,0.45)
```

> 理由：暗色背景下黑色阴影与背景融合度高，降低透明度（相比亮色模式增加约 2.5x 不透明度）以保持视觉深度层次。

### 9.2 暗色模式设计原则

1. **内容区对比度舒适**：正文色使用 #E8E8E8 而非纯白，避免亮色内容在深色背景上产生刺眼眩光。链接和交互元素适当提亮以弥补对比度感知差异。
2. **代码块背景更深**：代码块背景使用 `#0D0D0D`（比普通背景更深一级），代码文字色使用 `#D4D4D4`（比正文稍亮），保持代码区与非代码区清晰区分。
3. **标签色适配**：标签背景改为深色+亮文字，标签圆点缀点色保持不变。在笔记列表和编辑器中，标签的可读性和色彩区分度不低于亮色模式。
4. **搜索高亮**：亮色模式使用黄色背景（`#FEF3C7`）高亮搜索匹配文字，暗色模式下改为暗金色（`#3B3018`），匹配文字色不变。
5. **图片/插画**：暗色模式下，浅色图片添加半透明暗色覆盖层（`rgba(0,0,0,0.15)`），白色背景插画替换为深色背景版本。
6. **过渡动效**：亮色/暗色切换时，背景色和文字色使用 300ms ease-in-out 过渡动画，避免瞬间闪烁。
7. **跟随系统**：默认跟随系统暗色模式（`prefers-color-scheme: dark`），设置页提供手动切换选项（亮色/暗色/跟随系统）。

#### 搜索

1. **搜索集成**：搜索入口集成在 Navigation Bar 的 `.searchable` modifier，本机原生搜索体验
2. **搜索作用域**：可在笔记列表 Tab 中搜索全部笔记，或在特定笔记本内搜索
3. **搜索结果**：使用 `List` 展示，搜索结果按相关度排序，匹配文字使用系统高亮色
4. **搜索建议**：显示搜索历史、「在 XX 中搜索」等建议
5. **结果导航**：点击搜索结果 Push 到编辑器页面（NavigationLink）

#### 图谱

设计要点：
1. **全屏展示**：图谱在独立视图中全屏展示，隐藏 Tab Bar
2. **交互**：双指缩放 + 拖拽平移（使用 UIKit `UIPinchGestureRecognizer` + `UIPanGestureRecognizer` 或 SwiftUI `MagnificationGesture` + `DragGesture`）
3. **控制**：右上角缩放控件（+ / - / 适应屏幕），底部 Sheet 显示选中节点信息
4. **节点详情**：点击节点后，使用底部 Sheet（`.sheet(…)`）展示笔记标题、标签、反向链接数量、「打开笔记」按钮
5. **关闭**：左上角关闭按钮或左滑返回

#### 设置

设计要点：
1. **列表样式**：使用 iOS 原生 `List` 配合 `InsetGrouped` 样式
2. **分组 Section**：
   - **账户**：头像、昵称、邮箱
   - **同步**：同步状态、手动同步、最近同步时间
   - **编辑器**：字号、字体、Focus Mode 设置
   - **外观**：亮色/暗色/跟随系统
   - **数据**：导出（Markdown / JSON）、导入、清空数据
   - **关于**：版本号、隐私政策、反馈
3. **交互**：使用原生 NavigationLink 推入详细设置页，Toggle 开关使用原生 `.toggle()`，Picker 使用原生选择器

## 12. Android App 设计

### 12.1 Android 设计语言

- **设计规范**：遵循 Material Design 3 (Material You)
- **字体**：使用 Roboto / Google Sans（系统自带，无需引入）
- **组件风格**：
  - Top App Bar（大型/中型/小型）
  - Navigation Bar（底部导航）
  - Bottom Sheet（底部面板）
  - Navigation Drawer（导航抽屉）
  - FAB（Floating Action Button）
  - Material Card（卡片容器）
  - Preference（设置列表）
- **色彩系统**：Material 3 色彩方案（Primary / Secondary / Tertiary / Error / Surface / Outline）
- **动态取色**：Android 12+ 支持 Material You 动态取色（`monet`），自动从壁纸提取主题色
- **暗色模式**：跟随系统（`DayNight` 主题），使用 Material 3 暗色方案
- **导航模式**：Jetpack Navigation 组件，单 Activity 架构

设计要点：

2. **列表**：使用 Material 3 Card 样式（`elevated` / `filled`），卡片间间距 8px。圆角 `--radius-md`
3. **FAB**：右下角悬浮操作按钮（`FloatingActionButton`），主色 `Primary`，图标使用 `mdi-plus`
4. **Navigation Bar**：底部 4 个目的地，选中态使用 Primary 色，未选中使用 On Surface Variant

设计要点：
1. **Top App Bar**：中型标题样式，左侧返回箭头，中间显示笔记标题（可编辑），右侧保存状态图标（💾 或云同步图标）和更多菜单
2. **内联工具栏**：悬浮在键盘上方（使用 Compose `KeyboardAction` 或 `WindowInsets` 适配），包含 Bold / Italic / Heading / List / Link 等格式化按钮
3. **`[[` 触发**：输入 `[[` 时弹出 Bottom Sheet，展示笔记搜索结果列表
4. **`#` 触发**：输入 `#` 时弹出 Bottom Sheet，展示标签层级树
5. **Markdown 实时预览**：与桌面端一致的 Live Preview 体验
6. **自动保存指示**：Top App Bar 右侧显示保存状态图标（保存中 / 已保存 / 未保存）
7. **硬件键盘支持**：支持外接键盘快捷键（`Ctrl+N` 新建、`Ctrl+S` 保存等）

#### 搜索

设计要点：
1. **搜索触发**：点击 Top App Bar 的搜索图标进入搜索模式，Top App Bar 自动展开为搜索框
2. **搜索建议与历史**：搜索框下方显示搜索历史和热门搜索建议
3. **实时搜索**：输入字符 200ms debounce 后更新结果
4. **结果列表**：使用卡片列表展示搜索结果，匹配文字高亮
5. **结果导航**：点击结果 Push 到编辑器页面

#### 图谱

设计要点：
1. **全屏 Canvas**：使用 Compose Canvas 或 Custom View 渲染力导向图
2. **交互**：双指缩放（`pinchToZoom`）+ 拖拽平移（`drag`），Material 3 风格缩放控件覆盖层
3. **节点详情**：点击节点后使用 Bottom Sheet（`ModalBottomSheet`）展示笔记信息，包含标题、标签、链接数量、「打开笔记」按钮
4. **控制面板**：右上角缩放控件（+ / - / fit），底部搜索框高亮节点
5. **关闭**：返回键或左上角关闭按钮

#### 设置

设计要点：
1. **列表样式**：使用 Material 3 的 `Preference` 组件或 Compose `LazyColumn` + `Card` 构建
2. **分组 Section**（与 iOS 一致）：
   - 账户 / 同步 / 编辑器 / 外观 / 数据 / 关于
3. **交互**：设置项使用 Material 原生组件（Switch / RadioButton / Slider），分组 section 使用 Material 标题样式
4. **Navigation**：使用嵌套 Navigation（`NavHost`），设置子页面 Push 进入

## 13. 跨平台设计对照

### 13.1 三端关键差异

| 维度 | Web | iOS | Android |
|------|-----|-----|---------|
| **设计语言** | 自建 Design System | Apple Human Interface Guidelines | Material Design 3 (Material You) |
| **系统字体** | Noto Sans（引入） | SF Pro（系统自带） | Roboto / Google Sans（系统自带） |
| **图标库** | Material Design Icons (MDI) | SF Symbols（原生推荐） | Material Icons（原生推荐） |
| **主导航** | 侧边栏 240px（常驻） | Tab Bar 底部 4 Tab | Navigation Bar 底部 4 Tab |
| **新建笔记** | 顶部按钮 / 快捷键 `Cmd+N` | Navigation Bar 右上角 `+` | FAB 右下角悬浮按钮 |
| **搜索入口** | 顶部常驻搜索条（`Cmd+K`） | NavigationBar `.searchable` | Top App Bar 搜索图标展开 |
| **侧边栏** | 常驻 240px（桌面） | 无；使用 Tab + Sheet 组合 | Navigation Drawer 左侧拉出 |
| **暗色模式切换** | CSS 变量（`prefers-color-scheme`） | SwiftUI `@Environment(\.colorScheme)` | Material 3 DayNight 主题 |
| **列表样式** | 自定义列表项 | SwiftUI `List`（InsetGrouped） | RecyclerView / Compose LazyColumn（Card） |
| **编辑器工具栏** | 顶部浮动工具栏（自动淡出） | Keyboard Accessory View | 内联工具栏（键盘上方） |
| **`[[`/`#` 选择面板** | 下拉浮层 | `.sheet` 底部 Sheet | Modal Bottom Sheet |
| **手势返回** | 浏览器返回键 | 左侧边缘滑动返回（全屏手势） | 系统返回键 / 手势 |
| **下拉刷新** | 无（桌面端适用） | `.refreshable` modifier | SwipeRefreshLayout / pullRefresh |
| **离线存储方案** | IndexedDB / SQLite (via wasm) | Core Data / SQLite | Room Database / SQLite |
| **生物识别** | 不适用 | Face ID / Touch ID | 指纹 / 面部识别（BiometricPrompt）|
| **小组件** | 不适用 | WidgetKit 桌面小组件 | AppWidgetProvider 桌面小组件 |
| **分享集成** | Web Share API | Share Sheet 扩展 | Share Target（Intent Filter）|
| **分屏支持** | 浏览器自身管理 | Split View / Slide Over（iPad） | 分屏 / 多窗口自由 |
| **键盘快捷键** | 强大（`Cmd+N/S/K/Z` 等） | 仅外接键盘 | 仅外接键盘 |

### 13.2 跨平台一致性原则

| 原则 | 说明 |
|------|------|
| **功能一致性** | 三端核心功能（笔记 CRUD、Markdown 编辑、双向链接、标签、图谱）完全一致 |
| **交互差异化** | 导航模式、手势、列表样式遵循平台惯例，不强求统一 |
| **数据互通** | 同一账户下 Web/iOS/Android 数据通过云端同步，体验无缝 |
| **品牌色保留** | 三端统一使用蓝色作为品牌强调色。Android 端 Material You 可替换，但提供保留品牌色的选项 |
| **功能分级** | 平台特有功能（小组件、生物识别等）不阻碍跨平台核心体验对齐 |
| **命名一致** | 三端产品名称「MindFlow」统一，功能命名（笔记、标签、图谱、双向链接）统一 |
